# Temple of Nihility v1.0.0-beta.2

> 开发中测试版 / In-development beta

这是 `Temple of Nihility` 面向 **Minecraft 26.2 / NeoForge 26.2.0.7-beta** 的第二个公开测试版。
当前版本继续用于测试虚空宝库、无相能量、传送石碑联动、Curios 饰品、界面、贴图和世界生成结构；内容、数值、合成、结构和存档数据仍可能在后续版本调整。

## 目标环境

- Minecraft: `26.2`
- Loader: `NeoForge 26.2.0.7-beta`
- Java: `25`
- Mod: `1.0.0-beta.2`
- Jar: `temple-of-nihility-neoforge-26.2-1.0.0-beta.2.jar`
- SHA256: `7E50C71E9DBED128585046399D7FA4D87A7D2503A39F5E30B92ABF9038CFBCA4`

## 本版重点

- 新增并整理无相能量体系：无相汲能棱柱、无相能量容器、蓄能/传导饰品与装备能量联动。
- 新增虚空传输体系第一版：物品、流体和能量在相对方向之间周期性中继。
- 新增虚空传送石：Waystones 存在时可打开传送列表，取消不消耗能量，传送成功后消耗无相能量。
- 优化虚空宝库终端：滚动式仓储、搜索、排序、合成区、容量显示、保护开关和关闭常加载确认。
- 优化宝库多方块连接：水平连接和上下连接都有视觉变化，多个宝库会合并为一个网络。
- 修复祭坛仪式输出槽堵住时仍先扣无相能量的问题。
- 修复虚空终端空仓/无搜索结果没有文字提示的问题。
- 补齐方块 BlockItem 的中英文语言键，降低 JEI/创造栏显示翻译 key 的概率。
- 世界生成结构已启用测试，并改为使用结构生成流程传入的 `RandomSource`。
- README 和依赖说明同步当前真实状态：Curios 是必需依赖，JEI / Waystones 是可选联动。

## 已知说明

- 本版本仍是测试版，不建议直接用于长期主存档。
- NeoForge 26.2 本身仍是 beta 目标，API 与依赖生态仍可能变化。
- 虚空宝库、虚空玻璃和虚空水晶仍有半透明渲染策略待继续打磨，可能在部分角度出现视觉穿透或排序问题。
- 当前实例日志中的 `earth_online:*` 旧方块/物品 registry 缺失来自其他 mod/旧存档，不属于本模组本次发布修复范围。
