# Temple of Nihility v1.0.0-beta.1

> 开发中测试版 / In-development beta

这是 `Temple of Nihility` 面向 **Minecraft 26.2 / NeoForge 26.2.0.7-beta** 的首个公开测试版。
当前版本用于测试核心玩法、宝库网络、Curios/JEI 联动、虚无状态、界面与贴图方向；内容、数值、合成、结构和存档数据仍可能在后续版本调整。

## 目标环境

- Minecraft: `26.2`
- Loader: `NeoForge 26.2.0.7-beta`
- Java: `25`
- Mod: `1.0.0-beta.1`
- Jar: `temple-of-nihility-neoforge-26.2-1.0.0-beta.1.jar`
- SHA256: `7529DFF52A5D7EB21A957093CBF8CB9A0C41E101FE5CD5B60B3AA27F2F93001D`

## 主要内容

- 虚无材料、建筑方块、装备、工具、武器和功能道具。
- Curios 饰品联动，包括虚无终端、虚无灯核、复苏宝珠、磁核等。
- 虚无宝库多方块仓储网络：相邻宝库合并容量，终端支持搜索、排序、合成栏、保护与常加载设置。
- 虚无祭坛多方块仪式系统：根据结构等级进行材料进阶。
- 自定义虚无状态：
  - `遁入虚无`：完整虚无化，攻击会解除。
  - `融于虚无`：完整虚无化，持续到时间结束。
  - `隐入虚无`：弱化版，仅免疫环境伤害。
- JEI 可选联动：终端搜索与物品识别。

## 已知说明

- 本版本是测试版，不建议直接用于长期主存档。
- NeoForge 26.2 本身仍是 beta 目标，API 与依赖生态仍可能变化。
- Curios 是必需依赖；JEI 是客户端可选联动。
- 结构世界生成仍未恢复，避免 26.2 结构注册不稳定影响进世界。
